Naming conventions for different things in React
1. Naming Components
- PascalCase. Eg: ProductCard.jsx

2. Variables
- camelCase. Eg: const firstName = "Rohail"; 

3. Naming Files
- PascalCase. Should match the name of the component if used. Eg: ProductPage.jsx

4. Props
- Use a descriptive name. Eg 'user' instead of 'usr'

5. State variables
- Prefix state variables with is, has, or should to denote boolean values. Eg: [isActive, setIsActive]

6. Event handlers
- Use 'handle' as prefix and then a suitable name corresponding to the event. Eg: handleEventClick

7. CSS classes
- Use lowercase letters and hyphens. Eg: class='example-container'

8. Utility functions or helper functions
- Use camelCase for utility or helper functions. Eg. calculateDate();
